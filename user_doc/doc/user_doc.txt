 *  \section cfu-intro Introduction
 *
 *  crystal_facet_uml is a uml diagram drawing tool
 *  that creates a set of consistent uml diagrams.
 *
 *  If started in graphical mode, it shows a window with
 *  \li toolbar on top,
 *  \li drawing area in the center,
 *  \li element configuration widgets below and
 *  \li an optional notification bar at the bottom.
 *
 *  \image html screenshot_1.png
 *  \image latex screenshot_1.png "" width=15cm
 *
 *  Additionally, crystal_facet_uml can be started from command line
 *  to check and repair database files.
 *  Run
 *  \code{.sh}
 *  ./crystal_facet_uml -h
 *  \endcode
 *  to get a list of supported parameters.
 *
 *  \section cfu-tools Tool Bar
 *
 *  \image html main_window_sketch_1.png
 *  \image latex main_window_sketch_1.pdf "" width=3cm
 *
 *  \image html resources/file_use_db.png
 *  \image latex resources/file_use_db.pdf "" width=1cm
 *  \li Opens an existing database file or creates a new database file
 *
 *  \image html resources/file_export.png
 *  \image latex resources/file_export.pdf "" width=1cm
 *  \li Exports all diagrams to the selected folder (supported formats are txt, png, pdf, ps and svg)
 *
 *  \image html resources/file_new_window.png
 *  \image latex resources/file_new_window.pdf "" width=1cm
 *  \li Opens another window on the same database.
 *
 *  \image html resources/tool_create_diagram.png
 *  \image latex resources/tool_create_diagram.pdf "" width=1cm
 *  \li Create a new diagram
 *
 *  \image html resources/tool_navigate.png
 *  \image latex resources/tool_navigate.pdf "" width=1cm
 *  \li Navigate to parent or child diagrams
 *
 *  \image html resources/tool_edit.png
 *  \image latex resources/tool_edit.pdf "" width=1cm
 *  \li Modify elements in the diagram
 *
 *  \image html resources/tool_create_object.png
 *  \image latex resources/tool_create_object.pdf "" width=1cm
 *  \li Create elements in the diagram
 *
 *  \image html resources/edit_cut.png
 *  \image latex resources/edit_cut.pdf "" width=1cm
 *  \li Cut all pink-cornered elements to the clipboard (features not yet supported)
 *
 *  \image html resources/edit_copy.png
 *  \image latex resources/edit_copy.pdf "" width=1cm
 *  \li Copy all pink-cornered elements to the clipboard (features of classifiers are copied independantly of their corner-colors)
 *
 *  \image html resources/edit_paste.png
 *  \image latex resources/edit_paste.pdf "" width=1cm
 *  \li Pastes diagrams and classifiers from the clipboard to the uml model. (Relationships are not pasted)
 *      If id and name are identical to an existing element, an instance of the existing element is pasted to the diagram.
 *      Otherwise a new element is created.
 *
 *  \image html resources/edit_delete.png
 *  \image latex resources/edit_delete.pdf "" width=1cm
 *  \li Deletes all pink-cornered elements. This operation may fail if marked elements have unmarked children.
 *
 *  \image html resources/edit_instantiate.png
 *  \image latex resources/edit_instantiate.pdf "" width=1cm
 *  \li Toggles the pink-cornered classifiers between classes and instances. (Does not work for relationships and features)
 *
 *  \image html resources/edit_highlight.png
 *  \image latex resources/edit_highlight.pdf "" width=1cm
 *  \li Toggles the pink-cornered classifiers between yellow-marked, greyed-out and normal. (Does not work for relationships and features)
 *
 *  \image html resources/edit_reset.png
 *  \image latex resources/edit_reset.pdf "" width=1cm
 *  \li Resets the pink-cornered selection
 *
 *  \image html resources/edit_undo.png
 *  \image latex resources/edit_undo.pdf "" width=1cm
 *  \li Un-does the last operation (Opening a database and exporting files cannot be undone)
 *
 *  \image html resources/edit_redo.png
 *  \image latex resources/edit_redo.pdf "" width=1cm
 *  \li Re-does the last un-done operation
 *
 *  \image html resources/crystal_facet_uml.png
 *  \image latex crystal_facet_uml.pdf "" width=1cm
 *  \li Shows version, license and copyrights
 *
 *  \section cfu-draw Drawing Area
 *
 *  \image html main_window_sketch_2.png
 *  \image latex main_window_sketch_2.pdf "" width=3cm
 *
 *  \image html resources/tool_create_diagram.png
 *  \image latex resources/tool_create_diagram.pdf "" width=1cm
 *  \li To create a child-diagram, click into the children-area of the current view.
 *
 *  \image html resources/tool_navigate.png
 *  \image latex resources/tool_navigate.pdf "" width=1cm
 *  \li To navigate to the parent diagram, click on the parent diagram.
 *  \li To navigate to a child diagram, click on the child diagram.
 *  \li To restructure the diagram tree by shifting a child diagram up and the parent down, click on the child diagram and press F7.
 *
 *  \image html resources/tool_edit.png
 *  \image latex resources/tool_edit.pdf "" width=1cm
 *  \li To move classifiers within the diagram, press, drag and release the mouse button.
 *  \li To select the diagram or a classifier or a feature or a relationship with yellow corners, click on this object.
 *  \li To mark diagram and/or classifiers and/or features and/or relationships with pink corners, click on these objects twice.
 *
 *  \image html resources/tool_create_object.png
 *  \image latex resources/tool_create_object.pdf "" width=1cm
 *  \li To create a classifier, click at an empty space in the diagram.
 *  \li To create a child classifier, click into a classifier that can contain children (e.g. nodes, components, packages, activities).
 *  \li To create a relationship, press on the source classifier and drag it to the destination classifier.
 *  \li To create a feature, click into a classifier that can contain features (e.g. blocks, classes, objects, interfaces).
 *

 *
 *  \image html main_window_sketch_3.png
